### 2. Graph Transformer Architecture Discussion

Worked through the core concepts distinguishing GT from GCN, focusing on what makes transformers architecturally different and why that matters for interpretability experiments.

**Key concepts covered**:

**Sequential vs non-sequential computation graphs**:
- GCN: each layer independently aggregates from fixed neighbors, no state carried between layers beyond the output tensor
- GT: each block adds a residual update to a shared hidden state — the same vector accumulates contributions from every block in sequence

**Residual stream**:
- Each transformer block reads from a shared vector and writes an additive update back to it
- No block replaces its input; all blocks contribute incrementally to a single scratchpad
- This means early blocks shape the input that later blocks receive, creating cross-block dependencies

**Content-dependent attention routing**:
- Attention weights are computed from the current hidden state (query/key dot products), not from fixed graph topology alone
- The graph mask constrains *which* neighbors are eligible, but *how much* attention each eligible neighbor receives is determined by the hidden state at that layer
- Patching block 0's output changes the hidden state that block 1 reads — which can shift block 1's routing (who it attends to) in addition to changing what gets aggregated

**Why interventions don't compose cleanly**:
- In a linear model, ablating unit A and ablating unit B independently gives you the same result as ablating both together
- In a transformer, ablating block 0 changes the input to block 1, which changes block 1's behavior — the effects interact rather than add

**Three experiments identified** (in order of complexity):

| 1 | Residual stream isolation | Zero block 0 vs block 1 — which matters more for accuracy? |
| 2 | Composition test | Ablate head independently per block then jointly — do effects add up? |
| 3 | Attention routing shift | After patching block 0, does block 1's routing (who it attends to) change? |

### 3. Attention Routing Shift Experiment (Completed)

Implemented `compare_attention_routing_with_nnsight()` in `src/gcn_citation/models/gt_nnsight.py` and ran it end-to-end on Cora.

**Function signature**:
```python
def compare_attention_routing_with_nnsight(
    model,
    *,
    features,
    degree_features,
    attn_mask,
    patch_block_index: int = 0,
    observe_block_index: int = 1,
    top_k_nodes: int = 5,
) -> dict[str, object]:
```

**What the function does**:
- Run 1 (baseline): normal forward pass, captures block-1 attention weights with shape `[heads, nodes, nodes]`
- Run 2 (patched): zeros block-0 output via NNsight intervention, captures block-1 attention weights under same shape
- Computes per-head routing diff (L1 distance between baseline and patched attention distributions)
- For each head, identifies the `top_k_nodes` with largest routing change and reports whether their top-attended neighbor changed

**Standalone script**: Created `run_routing_shift.py` at project root — trains GT on Cora then immediately runs the routing shift experiment and prints results.

### 5. Key Findings and Interpretation

**Routing is mostly stable**: A mean shift of 0.0001 across all nodes means the vast majority of nodes barely change who they attend to when block 0 is zeroed out. Block 0's contribution to determining *routing* is small.

**Routing is anchored by the input projection**: The residual stream carries raw input features through from the very beginning. By block 1, the hidden state is dominated by the input projection output (which bypasses block 0 entirely via the residual path). This is sufficient to determine most attention patterns without block 0's contribution.

**A few nodes shift dramatically**: Max shift of 0.44 is large in absolute terms, and among the most-shifted nodes the majority completely changed their top-attended neighbor. These are nodes for which block 0's refinement was meaningful enough to flip routing decisions.

**Values matter more than routing**: 90% of predictions changed (2432 out of 2708 nodes) despite near-zero mean routing shift. Block 0's contribution to *what gets aggregated* (the value vectors) is critical even when routing (who gets attended to) is largely unchanged.

**Core asymmetry**: This is the central interpretability finding — routing (driven by query/key computations) is stable and anchored in the residual stream; value contribution is fragile and heavily dependent on each block's output. Ablating a block collapses its value contribution while leaving routing mostly intact.
