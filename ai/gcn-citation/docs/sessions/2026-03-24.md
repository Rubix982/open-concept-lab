# Session Summary: March 24, 2026

This note captures the main progress, findings, and next steps from today's graph learning session.

## What we built and clarified

- We deepened the conceptual understanding of:
  - GCNs
  - normalized adjacency matrices
  - gradient descent and manual backpropagation
  - t-SNE as an embedding inspection tool
  - transductive vs inductive graph learning
- We extended the project from the fixed Cora benchmark into a reusable arXiv pipeline:
  - category-based metadata fetching from arXiv
  - aggressive local caching
  - processed dataset snapshots
  - TF-IDF features from title + abstract
  - similarity-graph construction
- We introduced a model-family structure for future work:
  - `models/`
  - `pretraining/`
  - `prompting/`

## GCN findings

- On the cached arXiv similarity graph, shallower GCNs performed better than deeper ones.
- `depth_1` clearly outperformed deeper GCN variants both:
  - visually in t-SNE
  - quantitatively in validation and test accuracy
- This is consistent with over-smoothing on the current graph construction.

## GraphSAGE progress

We implemented GraphSAGE in stages:

### GraphSAGE v1

- mean aggregator
- fixed fanout
- full-batch sampled-neighborhood path

### GraphSAGE v2

- mini-batch GraphSAGE path
- target-node batching
- local sampled computation neighborhoods
- reusable CLI integration

### GraphSAGE v2.1

- cleaner batch/subgraph expansion internals
- richer diagnostics:
  - `num_batches_last_epoch`
  - `avg_target_nodes_per_batch`
  - `avg_local_nodes_per_batch`
  - `avg_sampled_links_per_batch`
  - `avg_layer_active_counts`
  - `avg_layer_expanded_counts`
  - `avg_epoch_seconds`
  - `total_training_seconds`

## GraphSAGE findings so far

- GraphSAGE is outperforming the current GCN setup on the arXiv corpus.
- `GraphSAGE v2` appears stronger than `GraphSAGE v1`.
- Across the current experiments, shallow depth remains strong:
  - `depth_1` often looks better than `depth_2`
- This suggests the batching-based GraphSAGE path is the right direction for larger graphs.

## Sampler work completed

We replaced the earlier redundant sampler setup with real distinct GraphSAGE sampler variants:

- `uniform`
- `with-replacement`
- `degree-weighted`

Important implementation note:

- `with-replacement` now correctly handles duplicate sampled neighbors by converting duplicate counts into aggregation weights instead of silently collapsing them.

Artifact layout is now sampler-aware:

- `artifacts/graphsage/<variant>/<sampler>/<mode>/...`
