# Design and HCI

Human-centered design in practice — how to build AI systems that actually serve the
people they affect. The sequence matters: design decisions made before the first line
of code determine almost everything about whether the system serves people or extracts
from them.

---

## [Honest Build Sequence](honest-build-sequence.md)
The correct sequence for building AI systems that serve people: Plan → Validate →
Evaluate → Then Build. The sequence almost nobody follows.

Phase 1 (Plan): detailed document answering what, for whom, with whose consent, governed
how, failing how, corrected how — detailed enough for a second person to execute independently.

Phase 2 (Validate): Wizard of Oz before the system exists. Show researchers mock outputs,
watch where comprehension breaks, note what they trust and question. The researcher
outreach designed for the Bau Lab project is simultaneously UX testing, quality validation,
and ethical consent.

Phase 3 (Evaluate): run through Canca's PiE model (core vs instrumental), the Venn diagram
(whose interests overlap), the Microsoft 18 guidelines (pre-flight check), Shneiderman's
two-dimensional framework (top-right quadrant check), Xu's AI-first UCD.

Phase 4 (Build): with right-kind uncertainty (technical, testable) not wrong-kind
uncertainty (whose interests does this serve — answered implicitly by whoever wrote
the first line of code).

Why race logic fails for the knowledge infrastructure: the moat is relationships not
code, consent cannot be retrofitted, wrong kind of speed is self-defeating. The months
of planning are the product.
